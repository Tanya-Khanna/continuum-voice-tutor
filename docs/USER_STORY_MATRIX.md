# Continuum v7 user-story proof matrix

The current 38 user stories are defined in `docs/BUILD_PLAN.md` and mapped to
automated, model, carrier, Mission Control, and judge evidence in
[`TESTING_GUIDE.md`](TESTING_GUIDE.md#12-current-v7-user-story-coverage).

Use that guide as the executable release ledger. It supersedes the historical
v6 matrix that referenced retired subject menus, placement, duration selection,
curriculum packs, guardian voice menus, and recurring outbound lesson calls.

The current product gate requires:

- Language first, then separate name and learner-code turns.
- One open question: `What would you like to learn?`
- No subject, grade, curriculum, mode, placement, or duration menu.
- Evidence-based diagnosis, method switching, teach-back, transfer, reflection,
  and honest understanding state.
- Speech, stage-aware DTMF, selective memory, exact drop/resume, and bounded SMS.
- One-time consented reminders; the recurring lesson-call scheduler stays off.
- Teacher-role child-safety boundaries, factual uncertainty, privacy, and trusted
  application control.
- Zero-credit checks, the revision-bound live GPT suite, the real carrier matrix,
  and five consecutive golden judge journeys on one commit.

The private carrier/submission receipt remains
`submission/release-input.example.json`; do not commit its completed copy.
